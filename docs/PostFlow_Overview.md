# PostFlow — планировщик Telegram-контента

> Пишешь один раз — публикуется по расписанию. Управление несколькими каналами, очередь, медиа, история, базовая аналитика.

---

## 1) Что такое PostFlow

PostFlow — SaaS-сервис для планирования постов в Telegram-каналах. Пользователь подключает **своего Telegram-бота** к каналу, пишет пост в редакторе, выбирает дату/время (с учётом таймзоны), и система отправляет публикацию автоматически через очередь.

Ключевая идея: **простая, понятная админка вместо “ботов-посредников”**, при этом контроль остаётся у пользователя (его bot token и его каналы).

---

## 2) Для кого

- **Creators / авторы каналов**: публикации в прайм-тайм, даже если офлайн.
- **SMM/агентства**: ведение нескольких каналов и календарь публикаций.
- **Малый бизнес**: регулярный контент без ручной рутины.
- **Контент-команды**: очередь публикаций + история + ответственность за результат (sent/failed).

## 3) MVP (что должно работать в первой версии)

MVP соответствует чеклистам `PostFlow_Front_CHECKLIST.md` и `PostFlow_Back_CHECKLIST.md`.

### Пользовательский сценарий

1) Вход/регистрация через Clerk → синхронизация на бекенде (`POST /auth/sync`).
2) Добавление Telegram-канала: `botToken + chatId` → проверка соединения → сохранение.
3) Создание поста: текст + (опционально) фото/видео → выбор канала → выбор времени → “Schedule” или “Send now”.
4) Пост попадает в очередь → воркер отправляет в Telegram → статус обновляется.
5) Пользователь видит историю, статусы, ошибки, ближайшие посты и базовые метрики.

### Функции MVP

- **Auth**: Clerk, защищённые роуты, `POST /auth/sync`.
- **Channels**: список/создание/удаление/verify; проверка владельца; **botToken хранится в зашифрованном виде**.
- **Posts**:
  - создание, обновление (только пока pending), удаление (с отменой job), send-now, duplicate
  - список с фильтрами, пагинация, поиск
  - stats для дашборда (scheduled/sent/failed)
- **Uploads**: загрузка медиа в Cloudflare R2 (jpg/png/gif/mp4, до 50MB), выдача публичного URL.
- **Scheduler**: BullMQ + Redis (delay по scheduledAt), retry/backoff, корректная обработка 429 (Retry-After).
- **UI**: layout с сайдбаром/хедером, тосты, skeletons, empty states, переключатель темы.

## 4) Техстек (как в чеклистах)

## 5) Бизнес-стратегия и монетизация

### Позиционирование

PostFlow — “**минималистичный, быстрый планировщик Telegram**” с фокусом на:

- надежность отправки (очередь + retries + статусы)
- хороший UX редактора и предпросмотра
- доступную цену для одиночных авторов и малого бизнеса

### Модель дохода (пример)

- **Free**: 1 канал, лимит постов/мес, базовые статусы/история
- **Pro**: несколько каналов, больше лимитов, расширенные фильтры/календарь, приоритет в очереди
- **Business**: команда (несколько пользователей), роли, расширенная аналитика, SLA/поддержка, лимиты выше

### Основные расходы (ориентиры)

Ниже — список “куда уходит деньги” и что будет расти вместе с пользователями:

- **Инфраструктура**:
  - Railway: бекенд + Postgres + Redis (рост от нагрузки/объёма данных)
  - Cloudflare R2: хранение медиа + egress (зависит от объёма и просмотров)
  - Vercel: фронт (обычно недорого на старте)
- **Auth/Безопасность**:
  - Clerk: платно при росте MAU/активных пользователей
  - секреты/шифрование: ключи, ротация, соблюдение минимальных практик безопасности
- **Операционка**:
  - домен, почта, мониторинг/логирование, алерты
  - поддержка пользователей (время/саппорт), документация, контент
- **Платежи** (когда подключатся): комиссия провайдера + возможные чарджбеки

## 6) Что можно делать после MVP

- **Командная работа**: роли (owner/editor/viewer), комментарии, согласование постов.
- **Шаблоны/рубрикатор**: библиотека шаблонов, “сетка контента”, повторяющиеся публикации.
- **Более умная аналитика**: частота публикаций, тайминг, успех/ошибки по каналам, “какие часы/дни лучше”.
- **Интеграции**: Notion/Google Docs, webhooks, Zapier/Make, экспорт/импорт контента.
- **Надёжность и контроль**: дедупликация задач, идемпотентность отправки, отчёты по delivery, улучшенная обработка Telegram rate limits.
- **Мультимедиа**: альбомы, несколько медиа, черновики, предпросмотр разных типов.

## 7) Дорожная карта

### Ближайшие 4–8 недель (MVP → первый запуск)

- закрыть все пункты фронт/бек чеклистов
- стабильный воркер, корректные статусы `pending/sent/failed`, retry/backoff
- базовая админка: channels + posts + editor + календарь + дашборд
- деплой front/back, health check, минимальная наблюдаемость

### 3–12 месяцев (рост и монетизация)

- подключить платежи, тарифы и лимиты
- улучшить аналитику (страница Analytics из чеклиста как “v1”)
- командные функции “лайт”: несколько пользователей на workspace
- маркетинг: контент/кейсы, партнерства с авторами/агентствами

### 1–5 лет (куда целиться)

Цель на 5 лет — стать **контент-операционной системой для Telegram**:

- планирование + согласование + библиотека контента + календарь + отчёты
- высоконадежная доставка (SLO/SLA, масштабирование очередей и воркеров)
- экосистема интеграций и API (чтобы PostFlow встраивался в процессы команд)
- “умные рекомендации” (тайминг/ритм публикаций/качество контента) как отдельная ценность
